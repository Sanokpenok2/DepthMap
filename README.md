# Карта глубины по стереопаре

Программа на Python строит карту глубины (диспарности) на основе двух
изображений стереопары — левого и правого. Используется алгоритм
Semi-Global Block Matching (SGBM) или классический Block Matching (BM)
из OpenCV, с опциональной пост-обработкой WLS-фильтром.

## Установка

```bash
pip install -r requirements.txt
```

## Веб-интерфейс

Простой GUI с загрузкой левого/правого фото и настройкой параметров:

```bash
python app.py
```

Откроется браузер на `http://127.0.0.1:7860`. Загрузите стереопару,
при необходимости файл `.npz` от калибровки и нажмите «Построить».
Клик по карте диспаритета показывает расстояние (нужна калибровка или focal/baseline).

## Трекинг объекта по SBS-видео

Скрипт `video_track_depth.py` берёт **одно SBS-видео** (side-by-side): левая
половина кадра — левая камера, правая половина — правая. Даёт выбрать объект
на первом кадре, сопровождает его трекером и считает расстояние по медиане
диспаритета внутри ROI.

```bash
python video_track_depth.py ^
    --video stereo_sbs.mp4 ^
    --calib stereo_calib.npz
```

Полезно для ускорения:

```bash
python video_track_depth.py ^
    --video stereo_sbs.mp4 ^
    --calib stereo_calib.npz ^
    --sgbm-interval 3 --tracker kcf --smooth 5 ^
    --threads 0 --workers 2 --async-sgbm
```

| Параметр | Описание | По умолчанию |
|---|---|---|
| `--video` | SBS-файл (левая половина — левая камера) | — |
| `--swap-lr` | Поменять половины местами | выкл. |
| `--calib` | Калибровка `.npz` (обязательна, кроме `--track-only`) | — |
| `--tracker` | `csrt`, `kcf` или `mosse` | `csrt` |
| `--roi-smooth` | Сглаживание рамки `[0..1)`: больше = плавнее (больше лаг) | `0.3` |
| `--lock-size` / `--no-lock-size` | Фиксировать размер рамки (резкое сжатие → LOST) | вкл. |
| `--keep-aspect` / `--no-keep-aspect` | Масштабировать рамку равномерно, сохраняя пропорции | вкл. |
| `--max-scale-step` | Макс. изменение масштаба рамки за кадр (0 = без лимита) | `0.05` |
| `--max-size-ratio` | Макс. изменение площади за кадр; иначе объявляется потеря | `1.6` |
| `--verify` / `--no-verify` | Отклонять дрейф рамки на другой объект/фон | вкл. |
| `--verify-threshold` | Мин. сходство с эталоном `[0..1]` | `0.45` |
| `--verify-rel` | Отклонять, если score &lt; EMA×verify-rel | `0.75` |
| `--min-iou` | Мин. IoU с предыдущей рамкой (анти-перескок) | `0.35` |
| `--max-jump` | Макс. прыжок центра за кадр (доли диагонали ROI) | `1.0` |
| `--lost-patience` | Плохих кадров подряд до объявления потери | `2` |
| `--reacquire` / `--no-reacquire` | Повторный захват объекта после потери | вкл. |
| `--reacquire-threshold` | Порог совпадения для перезахвата `[0..1]` | `0.62` |
| `--reacquire-radius` | Окно поиска при перезахвате (доли размера ROI) | `2.5` |
| `--reacquire-global` | Искать по всему кадру (может хватать похожий фон) | выкл. |
| `--reacquire-interval` | Поиск при потере каждые N кадров (больше = выше FPS) | `5` |
| `--reacquire-scale-min` | Мин. масштаб эталона (меньше — искать более мелкий/далёкий объект) | `0.35` |
| `--reacquire-scale-max` | Макс. масштаб эталона при перезахвате | `1.4` |
| `--click-tolerance` | Допуск цвета при выборе кликом (C) | `16` |
| `--no-grabcut` | Не уточнять границы объекта GrabCut'ом | выкл. |
| `--sgbm-interval` | Считать карту глубины каждые N кадров | `2` |
| `--smooth` | Медиана по последним N измерениям | `5` |
| `--threads` | Потоки OpenCV для SGBM/remap (`0` = все ядра) | `0` |
| `--workers` | Параллельная подготовка L/R кадров | `2` |
| `--async-sgbm` / `--no-async-sgbm` | SGBM в фоне, трекинг не ждёт | вкл. |
| `--track-only` | Только захват и трекинг, без SGBM и расстояния (калибровка не нужна) | выкл. |
| `--output` | Сохранить видео с оверлеем | — |

Управление в окне: **Space** — пауза, **R** — выбрать объект **рамкой**, **C** — выбрать объект **кликом** (границы определяются автоматически), **Q** — выход. Объект можно не выбирать на старте — видео проигрывается, а трекинг начинается после первого выбора. Выбор доступен в любой момент.

**Защита от дрейфа.** CSRT/KCF иногда не сообщают «lost», а перескакивают на
похожий/более текстурированный сосед (куст и т.п.), особенно когда границы
слегка пересекаются. Проверка `--verify` отклоняет кадр, если:
сходство с эталоном низкое, score упал относительно недавнего среднего
(`--verify-rel`), IoU с предыдущей рамкой слишком мал (`--min-iou`), прыжок
центра велик (`--max-jump`), либо на старой позиции объект всё ещё узнаётся
лучше, чем на новой. При отклонении трекер откатывается на последнюю хорошую
позицию; после `--lost-patience` плохих кадров объявляется потеря.

**Повторный захват после потери.** Поиск эталона идёт **локально** вокруг
последней позиции (`--reacquire-radius`), чтобы не хватать похожий статичный
объект в другом конце кадра. Полный кадр — только с `--reacquire-global`.
Чтобы не ронять FPS, при `LOST` не гоняется CSRT/SGBM каждый кадр: поиск
выполняется раз в `--reacquire-interval` кадров (по умолчанию 5) на уменьшенном
разрешении, в диапазоне масштабов `--reacquire-scale-min`…`--reacquire-scale-max`
(по умолчанию 0.35…1.4 — чтобы находить объект, который стал заметно меньше после
удаления). Окно поиска постепенно расширяется, пока цель потеряна. Порог —
`--reacquire-threshold`, отключается через `--no-reacquire`.

**Выбор кликом (C).** Клик по объекту → границы определяются автоматически
(`floodFill` по цвету + уточнение `GrabCut`). Чувствительность — `--click-tolerance`,
уточнение можно отключить `--no-grabcut`.
Разрешение кадров подгоняется под размер из калибровки с сохранением пропорций кадр↔калибровка через `resize`.

### Изолированный модуль трекинга

Логика захвата и сопровождения объекта вынесена в `object_tracker.py`
(класс `ObjectTracker` + помощники `select_object_roi`, `clamp_roi`). Модуль
не зависит от стерео, поэтому трекинг можно протестировать отдельно на **одном**
видео:

```bash
python object_tracker.py --video left.mp4 --tracker csrt
python object_tracker.py --video left.mp4 --smooth 0.3 --lock-size
python object_tracker.py --video left.mp4 --roi 100 80 60 60 --no-show --max-frames 50
```

В окне: **R** — выбор рамкой, **C** — выбор кликом (авто-границы), **Q** — выход.

Рамка сглаживается EMA-фильтром (`--smooth`/`--roi-smooth`): центр — по EMA
(при быстром смещении сглаживание ослабляется, чтобы не отставать),
размер — единым масштабом с сохранением исходных пропорций (`--keep-aspect`),
чтобы при удалении объекта рамка уменьшалась равномерно и не обрезала объект.
Резкие скачки масштаба ограничиваются `--max-scale-step`. `--lock-size` фиксирует
размер во время трекинга; после уверенного перезахвата размер обновляется
(объект мог стать меньше после удаления). Повторный захват —
`--reacquire` (порог `--reacquire-threshold`, масштабы `--reacquire-scale-min/max`).

Программно:

```python
from object_tracker import ObjectTracker

trk = ObjectTracker(kind="csrt")
trk.init(frame_bgr, (x, y, w, h))
ok, roi = trk.update(next_frame)
```

Внутри `video_track_depth.py` этот же класс используется для трекинга, а флаг
`--track-only` позволяет прогнать только его (без расчёта карты глубины).

## Использование

Базовый запуск:

```bash
python depth_map.py --left left.png --right right.png --output disparity.png
```

С WLS-фильтром (сглаживание, заполнение пропусков) и просмотром результата:

```bash
python depth_map.py -l left.png -r right.png --method sgbm --wls --show
```

Сохранить сырую карту диспаритетов для дальнейших расчётов:

```bash
python depth_map.py -l left.png -r right.png --save-raw disparity.npy
```

## Измерение расстояния до объекта

Расстояние вычисляется по диспаритету двумя способами:

- **С калибровкой** — через матрицу `Q` (результат в единицах `--square-size`,
  обычно мм):

```bash
python depth_map.py -l left.png -r right.png --calib stereo_calib.npz \
    --measure 320 240
```

- **Без калибровки** — по формуле `depth = focal · baseline / disparity`,
  если известны фокусное расстояние (в пикселях) и база камер (в мм):

```bash
python depth_map.py -l left.png -r right.png --focal 700 --baseline 60 \
    --measure 105 105 --measure 275 165
```

Диспаритет усредняется (медиана) по окну `--measure-window` вокруг точки,
чтобы снизить влияние шума. Отмеченные точки и подписи с расстоянием
сохраняются на выходной карте. В режиме `--show` можно **кликать мышью**
по карте диспаритета — расстояние в точке печатается и подписывается сразу.

## Калибровка и ректификация камер

Если снимки сделаны обычной (некалиброванной) стереопарой, изображения
нужно сначала **откалибровать и ректифицировать**. Для этого снимите
10–20 пар кадров шахматной доски с разных ракурсов обеими камерами.

Понадобится распечатанный шахматный паттерн (например 9×6 внутренних
углов). Печатайте в масштабе 100% и после печати проверьте размер клетки
линейкой — это значение и укажите в `--square-size`.

1. Выполните стереокалибровку:

```bash
python calibrate_stereo.py --left "calib/left_*.png" --right "calib/right_*.png" --cols 9 --rows 6 --square-size 25 --output stereo_calib.npz
```

Здесь `--cols`/`--rows` — число **внутренних** углов доски, `--square-size` —
размер клетки в мм (задаёт масштаб глубины). Программа выведет RMS-ошибку
(хорошо < 1 пикселя), рассчитанные `focal` и `baseline`, и сохранит все
параметры (`Q`, карты ремаппинга, `focal_px`, `baseline_mm` и т.д.).

Параметр `--alpha` (0..1) управляет обрезкой при ректификации:
- `1` (по умолчанию) — сохраняется весь кадр, по краям могут появиться чёрные поля;
- `0` — обрезка до общей валидной области, без чёрных полей, но заметно «уже» кадр.

После калибровки программа выводит проверку качества (RMS, база, % валидных
пикселей) и предупреждения, если калибровка подозрительная. Те же
предупреждения показываются в `depth_map.py` и веб-интерфейсе при загрузке
`.npz`.

Если после ректификации изображение кажется сильно обрезанным, пересоздайте
калибровку с `--alpha 1` (или промежуточным значением, например `0.5`).

2. Постройте карту глубины с ректификацией и получите глубину/облако точек:

```bash
python depth_map.py -l left.png -r right.png --calib stereo_calib.npz \
    --wls --depth depth.npy --point-cloud cloud.ply -o disparity.png
```

Файл `cloud.ply` можно открыть в MeshLab, CloudCompare или Blender.

## Важно

- Изображения стереопары должны быть **ректифицированы** (выровнены по
  горизонтали): соответствующие точки лежат на одной строке. Если
  снимки не выровнены, используйте `calibrate_stereo.py` и передайте
  результат в `--calib` (см. раздел «Калибровка и ректификация камер»).
- Файл калибровки строится под конкретное разрешение камер — стереопара
  для `depth_map.py --calib` должна иметь то же разрешение, что и кадры
  доски при калибровке.
- Чем ближе объект, тем больше диспаритет и тем "теплее" (ближе к
  красному в палитре JET) он отображается.
- Для перевода диспаритета в реальную глубину нужна фокусное расстояние
  `f` и база `B`: `depth = f * B / disparity`.

## Как это работает

1. Изображения загружаются в градациях серого.
2. Алгоритм сопоставления ищет для каждого пикселя левого изображения
   соответствующий пиксель в правом и вычисляет смещение (диспаритет).
3. При включённом WLS-фильтре строится дополнительная карта справа-налево,
   и результат сглаживается с учётом границ исходного изображения.
4. Карта диспаритетов нормализуется и раскрашивается выбранной палитрой.
